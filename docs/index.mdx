`browser-agent` drives a real Chrome browser with a large language model. You give
it a task in plain language; it observes the page, decides the next action, runs
that action over the Chrome DevTools Protocol (CDP), and repeats until the task is
done.

## The loop

```text
task ──▶ snapshot ──▶ LLM decision ──▶ action ──▶ snapshot ──▶ … ──▶ done
```

Every step the agent serializes the live DOM into a compact snapshot, asks the
model for the next action(s), executes them against the active page, and captures
a fresh snapshot. The run ends when the model emits a terminal `done` action — or
when a guardrail (timeout, max-failures, loop detection, budget) stops it.

## What makes it different

- **Raw CDP, real Chrome** — headed or headless, not a synthetic emulator.
- **Bring any model** — OpenAI, Anthropic, or local CLIs (Codex, Claude) via a
  pluggable transport layer.
- **Resilient by default** — self-healing actions, loop detection, post-condition
  assertions, and consecutive-failure limits.
- **Production observability** — structured run reports, OpenTelemetry export,
  JUnit XML, cost/budget tracking, and replay traces.
