A **transport** is the mechanism the agent uses to reach an LLM. There are three kinds:

| Transport   | What                                                                                                                            |
| ----------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `sdk-agent` | A vendor's Agent SDK (e.g. `@anthropic-ai/claude-agent-sdk`, `@openai/codex-sdk`). Best when available — runs in-process, no spawn cost. |
| `sdk-api`   | The vendor's plain REST SDK (`openai`, `@anthropic-ai/sdk`). Always available given an API key.                              |
| `cli`       | The vendor's CLI binary (`codex`, `claude`). Used when the SDK isn't viable but a logged-in CLI is.                          |

## Resolution flow

`resolveTransport()` picks the best transport for the current environment. It's called automatically by `createDecide()`.

```
detectEnv()                       → local | cloud
        │
provider × env → ordered chain    │
        ▼
for each candidate:
  probeTransport()        → ok? reason?
      sdk-agent: explicit API key, env API key, OR auth file present
      cli:       binary on PATH
      sdk-api:   explicit API key OR env API key present
        │
   first ok → buildDecide()
        ▼
                       { decide, resolution }
```

### Probe conditions

A candidate transport is only chosen if its probe passes:

- **`sdk-agent`** — an explicit `apiKey` option, the matching API-key env var, or a vendor auth file on disk (e.g. `~/.claude/.credentials.json`).
- **`cli`** — the vendor binary is found on `PATH`.
- **`sdk-api`** — an explicit `apiKey` option or the matching API-key env var.

Each rejected candidate is logged to stderr as `transport_unavailable` before the resolver moves on.

## Priority chain

The order candidates are tried depends on the provider and the detected environment:

| Provider    | env=local                       | env=cloud       |
| ----------- | ------------------------------- | --------------- |
| `codex`     | `sdk-agent` → `cli`             | (none — throws) |
| `claude`    | `sdk-agent` → `cli` → `sdk-api` | `sdk-api`       |
| `openai`    | `sdk-api`                       | `sdk-api`       |
| `anthropic` | `sdk-api`                       | `sdk-api`       |

Cloud disables the CLI and agent-SDK paths because they assume binaries and auth files on disk that won't exist in a typical container.

## Detecting cloud

`detectEnv()` resolves the environment in priority order:

<Steps>

### Explicit override

An explicit `env: "local" | "cloud"` option wins over everything else.

### Env var

`BROWSER_AGENT_ENV=local|cloud`.

### Cloud markers

Common platform signals: `KUBERNETES_SERVICE_HOST`, `AWS_LAMBDA_FUNCTION_NAME`, Vercel, Fly, Railway, and Cloud Run (`GOOGLE_CLOUD_PROJECT` + `K_SERVICE`).

### Default

`local`.

</Steps>

## Forcing a specific transport

```ts
import { createDecide } from "@peteqian/browser-agent-sdk";

const { decide, resolution } = createDecide({
  provider: "claude",
  transport: "sdk-api", // try only this transport
});
```

A forced transport still runs its probe. Forcing `sdk-api` still requires an explicit `apiKey` option or the matching API-key env var — forcing doesn't bypass the credential check.

Useful when you want deterministic behavior in CI, want to bypass a misbehaving CLI without uninstalling it, or are testing a specific path.

## Probing without running

```sh
browser-agent --probe --provider claude
```

Prints the resolution as JSON:

```json
{
  "provider": "claude",
  "env": "local",
  "transport": "cli",
  "fallbackFrom": "sdk-agent",
  "fallbackReason": "claude sdk-agent: no ANTHROPIC_API_KEY or ~/.claude/.credentials.json",
  "durationMs": 9
}
```

## Observing resolution at runtime

Pass `transportResolution` to `runTask` and a `transport_resolved` event fires before step 1:

```ts
import { createDecide, runTask } from "@peteqian/browser-agent-sdk";

const { decide, resolution } = createDecide({ provider: "codex" });

await runTask({
  task: "...",
  getNextAction: decide,
  transportResolution: resolution,
  onEvent: (event) => {
    if (event.type === "transport_resolved") {
      console.log(event.resolution);
    }
  },
});
```

See [Events](/docs/guides/events) for the full event stream.

## Logs

`resolveTransport()` writes structured JSON to stderr for both the chosen transport and every rejected fallback:

```
{"event":"browser_agent.transport_unavailable","provider":"claude","transport":"sdk-agent","reason":"..."}
{"event":"browser_agent.transport_resolved","provider":"claude","env":"local","transport":"cli","fallbackFrom":"sdk-agent"}
```

Pipe stderr through your structured logger of choice.
